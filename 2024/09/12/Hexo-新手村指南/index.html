Hexo 新手村指南

本文主要讲解利用 GitHub Page 服务的 Hexo 博客搭建指南。

在开始之前

GitHub

首先，由于我们需要使用到 GitHub Page 服务，所以，在此之前，我们需要有一个 GitHub 账号。

GitHub Page 相关

  1. 是什么?

    GitHub Pages 是一个由 GitHub 提供的静态网站托管服务，它允许用户利用 GitHub 仓库来托管和发布个人、项目或组织的网站。

  2. 为什么选择？

    • 使用简单
    • 自定义域名
    • 支持多种静态网站生成工具
    • 自动构建

  3. 如何运转？

  • 用户将 HTML、CSS和JavaScript文件储存在 特定的仓库[^1]中。
  • 仓库自动构建网站并发布。

Git

需要 Git 以实现 本地仓库 于 远程仓库 的链接。

需要注意的是——

Git 通常有两种连接方式，此处推荐第一种，因为其更适用于频繁的连接操作。

  1. SSH 连接[^2]
  2. HTTPS 连接

Node.js

用于 Hexo 环境的搭建。

安装方式[^3]

小试牛刀

目标：安装Hexo，并实现本地调试。

建议参考的文档—— Hexo Docs - Set Up

  1. 安装Hexo
npm install -g hexo-cli
  1. 新建项目的npm环境
hexo init <folder>
cd <folder>
npm install
  1. 启动服务器
hexo server

如果一切正常，终端会产生这样的输出——

INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.

恭喜你！！进度条过半，可以准备开香槟了。

这提示你通过http://localhost:4000/访问你的本地网页，并且你可以通过按键盘上的Ctrl+C停止本地服务。

再迈一个台阶

目标：实现GitPage配置与搭建

  1. 安装部署插件hexo-deployer-git
npm install hexo-deployer-git --save
  1. 更改_config.yml 文件中配置部署部分
deploy:
  type: git
  repo: <repository url>
  branch: [branch]

注意：

  • repo : 填写仓库的 SSH 链接。
  • branch : 一般选择 main 分支。
  1. 开始部署
hexo deploy

你可以在 GitHub 上看到网站的部署进度。

恭喜你，现在，你可以访问属于你的GitHub Page了！！
