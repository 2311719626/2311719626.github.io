Vue Step 1
2024/10/25

  • Vue 指令

    • 定义 Vue根据不同的指令，实现不同的功能。指令是带有 v-前缀的 特殊 标签属性。

    • 种类

      • v-html

        • 作用 设置元素的 inner HTML

        • 语法 v-HTML = “表达式”

      • v-show

        • 作用 控制元素显示隐藏

        • 底层原理 通过切换 CSS的display: none来控制显示隐藏

        • 语法 v-show = “表达式”

        • 场景 频繁切换显示隐藏的场景

      • v-if

        • 作用 控制元素是否渲染

        • 底层原理 通过判断条件控制元素的创建和移除（条件渲染）

        • 语法 v-if = “表达式”

        • 场景 要么显示要么隐藏，不频繁切换的场景

      • v-else和v-else-if

        • 作用 辅助 v-if 进行判断渲染

      • v-on

        • 作用 注册事件（①添加监听 ②提供处理逻辑）

        • 语法 ①v-on:事件名 = “内联语句” ②v-on:事件名 = “methods中的函数名”

        • 补充

          • 简写 @事件名=”函数”

          • 调用传参 向 函数 中传入参数

      • v-bind

        • 作用 动态设置 html的标签属性 -> src url title

        • 语法 v-bind:属性名=”表达式”

        • 补充

          • 简写 :属性名=”表达式”

      • v-for

        • 作用 基于数据循环，多次渲染整个元素

        • 范围 ①数组 ②对象 ③数字

        • 语法 v-for=”(item, index) in array”

        • 补充

          • 内容的填充使用

          • 如果不需要 index，可以直接简写指令 item in array

          • key 是给元素添加的唯一标识

      • v-model

        • 作用 给表单元素使用，实现双向数据绑定 -> 快速 获取 或 设置 表单元素内容

          • ① 数据变化 -> 视图自动更新

          • ② 视图变化 -> 数据自动更新

        • 语法 v-model=”data”

  • 训练案例

    • A 小黑的书架

      • 目标 实现数据的查看与删除

      • 解决思路

        • 通过 v-for 在 li中 呈现数据

        • 通过 @click在 button下 注册点击事件

        • 通过 item.id向调用函数传入参数

        • 函数中通过 filter()方法按 id 筛选出剩余的书本对象

        • 将获得的新对象通过 = 赋值给原数组对象，实现数据的删除

    • B 小黑记事本

      • 目标 实现任务管理（删除、增加、统计）

      • 解决思路

        • ① 渲染与删除

          • 通过 v-for 实现列表渲染

          • 通过 @click绑定 button绑定删除按钮

          • 通过 item.id传参 并 通过 filter() 方法实现删除后的数据更新

        • ② 添加

          • 通过 v-model 绑定输入框 -> 实时获取表单元素的内容

          • 通过 @click 绑定 button 绑定增加按钮

          • 在 add() 函数通过 trim() 方法写入验证

          • 通过 this.list.unshift({id:+new Date(), name:this.newItem})写入新数据

          • 将数据框内容清空

        • ③ 底部统计与清空

          • 统计通过 list.length 实现

          • 情况通过 list=[] 实现

          • 控制底部展示使用 v-show

描述列表

HTML 中的描述列表 (<dl>) 是一种用于表示术语及其定义或描述的列表。描述列表由以下三个标签组成：

  1. <dl>：定义描述列表的开始，该标签包含术语及其描述。
  2. <dt>：定义列表中的术语（definition term）。
  3. <dd>：定义列表中的描述（definition description）。
    以下是一个描述列表的基本结构示例：
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言（HyperText Markup Language）的缩写，是构建网页和其他信息可以在网页浏览器中查看的内容的标准标记语言。</dd>
  <dt>CSS</dt>
  <dd>层叠样式表（Cascading Style Sheets）的缩写，是一种用来表现HTML或XML文档的样式的计算机语言。</dd>
  <dt>JavaScript</dt>
  <dd>一种高级的、解释执行的编程语言，广泛用于网页开发，可以用来实现复杂的功能。</dd>
</dl>

什么是元信息

定义

元信息是关于数据的数据，描述了数据的基本属性和特征，但不包含数据内容本身。

以下为元信息的一些例子——

  1. 文件元信息：在计算机文件中，元信息可能包括创建日期、文件大小、作者、最后修改时间、文件类型等。
  2. 数据库元信息：在数据库中，元信息可以描述表的结构、字段类型、索引、关系、权限设置等。
  3. 网络元信息：在互联网上，网页的元信息可能包括关键词、描述、作者、最后更新时间等。

HTML 中的 <meta>元素

拓展 2——主流的编码方案

在计算机系统中，编码方案用于将人类可读的文本转换为计算机可以存储和处理的格式。以下是一些主流的编码方案：

  1. ASCII (American Standard Code for Information Interchange): 这是最早的编码方案之一，它使用7位来表示128个不同的字符，包括英文字母、数字和一些特殊符号。

  2. ISO-8859-1 (Latin-1): 这是对ASCII的扩展，使用8位来表示256个字符，它包括了西欧语言中使用的字符。

  3. UTF-8 (Unicode Transformation Format - 8-bit): UTF-8是一种可变长度的Unicode编码，它可以使用1到4个字节来表示一个字符。它向后兼容ASCII，并且在处理多语言内容时非常高效。

  4. UTF-16 (Unicode Transformation Format - 16-bit): UTF-16使用2或4个字节来表示一个字符。它几乎可以表示所有的Unicode字符，但由于它可能需要4个字节来表示某些字符，所以不如UTF-8那样空间高效。

  5. UTF-32 (Unicode Transformation Format - 32-bit): UTF-32使用固定的4个字节来表示每个Unicode字符。这种方式简化了字符的处理，但相比UTF-8和UTF-16，它占用的空间更多。

  6. GB2312/GBK/GB18030: 这些是中国大陆使用的编码方案，用于表示简体中文字符。GB18030是最新的标准，它向后兼容GB2312和GBK，并且包括了所有Unicode字符。

    这些编码方案中，UTF-8因其兼容性和对多语言的支持而成为互联网上最广泛使用的编码方案。随着全球化的发展，UTF-8正逐渐成为文本编码的默认标准。

GitHub 密钥配置

关于密钥

声明：此部分为 AIGC 内容。

SSH 密钥对包括两个密钥：公钥（Public Key）和私钥（Private Key）。它们在功能和使用上有以下主要区别：

公钥（Public Key）：

  • 分发：公钥可以安全地公开分享，不需要保密。
  • 功能：公钥主要用于加密数据，这样只有对应的私钥才能解密。在 SSH 中，它还用于验证通过私钥签名的数据。
  • 大小：公钥通常比私钥大。
  • 用途：在 SSH 连接中，公钥存储在服务器上，用于验证尝试连接的客户端是否拥有匹配的私钥。

私钥（Private Key）：

  • 保密：私钥必须保密，不应该与任何人分享，也不应该存储在公共位置。
  • 功能：私钥用于解密由公钥加密的数据，以及在 SSH 连接中证明身份，通过签名操作来验证数据的完整性和来源。
  • 大小：私钥通常比公钥小。
  • 用途：在 SSH 连接中，私钥存储在客户端计算机上，用于证明客户端的身份，并解密由服务器的公钥加密的数据。

主要区别总结：

  • 分发与保密：公钥可以公开，私钥必须保密。
  • 加密与解密：公钥用于加密数据，私钥用于解密。
  • 身份验证：在 SSH 中，公钥用于验证私钥的所有者，私钥用于证明身份。
  • 签名与验证：私钥用于对数据进行数字签名，公钥用于验证签名的有效性。

了解这些区别对于安全地使用 SSH 密钥非常重要。如果私钥泄露，那么任何拥有该私钥的人都可以伪装成密钥的所有者，访问使用该公钥的任何服务。因此，保护私钥的安全是使用 SSH 密钥进行安全通信的关键。

生成密钥

对于Linux 、 Windows系统均适用。

  • 打开终端。
  • 输入 ssh-keygen -t rsa -b 4096 -C "your_email@example.com" [^1]

配置密钥

  1. 找到生成的公钥文件。默认情况下，它的路径通常是：

    • Linux/macOS: ~/.ssh/id_rsa.pub

    • Windows: C:\Users\YourUsername\.ssh\id_rsa.pub

  2. 打开文件，复制里面的全部内容，在Github里配置。

测试SSH连接

在终端键入以下代码——

ssh -T git@github.com
